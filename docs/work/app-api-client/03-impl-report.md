## Deviations from the plan

- **Step 8 (R7) — test tool changed from msw to a `global.fetch` mock (ESCALATION, resolved by the orchestrator in full-auto).** The implementer found the plan-mandated `msw` test cannot run under `jest-expo@54` + pnpm and raised an ESCALATION rather than ship a fragile workaround. Root cause, established empirically: `import { http } from 'msw'` transitively pulls msw's ESM-only deps (`rettime`, `@open-draft/deferred-promise`, …); jest-expo selects their `.mjs` entry, but pnpm nests real packages at `node_modules/.pnpm/<pkg>/node_modules/<pkg>/…` and the **inner** `/node_modules/` re-triggers jest-expo's default `transformIgnorePatterns` negative lookahead, so the `.pnpm` whitelist never reaches them → `SyntaxError: Unexpected token 'export'`. The only fixes live in the shared `app/package.json` jest block, which the plan ruled off-limits.
  - **Resolution (orchestrator decision):** for unit-testing a thin `fetch` wrapper, mock `global.fetch` directly instead of intercepting at the network layer with msw. This is lighter, idiomatic for this layer, keeps the shared jest config untouched, and avoids the same wall on every future app-side test. It sets the app-side unit-test convention — **mock the `fetch`/service layer, not msw** (msw remains available for higher-level integration tests if ever needed) — a justified deviation from A4's msw mention for this test class.
  - **Result:** `api.test.ts` now mocks `global.fetch`, retains all six assertions (JWT injected / header omitted / 2xx parsed-typed / schema-mismatch → `ZodError` / error envelope → `ApiError` / 401 → `onUnauthorized` + reject), and passes: `pnpm --filter './app' run test` → **8/8 green** (6 api + 2 theme). `tsc` and `eslint` clean. The production module `app/core/services/api.ts` was unchanged by this resolution.

- **Minor, within scope:** added a private `ApiConfigError` class (used by `resolveBaseUrl`) so the "env unset" failure is a typed throw rather than a bare `Error`; this is an implementation detail of plan step 3 (R2), not a new requirement.
