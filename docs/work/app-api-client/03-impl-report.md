## Deviations from the plan

- **Step 8 (R7) — the `msw` test is authored but does not run under jest-expo.** The plan's Risks section anticipated "msw / jsdom fetch wiring flakiness" and instructed IMPLEMENT to "wire it locally in the test file … do **not** touch the shared `app/package.json` jest block." That local-only remedy is **not achievable** with the installed `msw@2.14.6` under `jest-expo@54` + pnpm. Root cause, established empirically:
  - `import { http } from 'msw'` (the barrel) transitively `require`s msw's WebSocket subsystem (`./ws`), which pulls a chain of dependencies.
  - jest-expo's resolver selects the **ESM** (`.mjs` / `module`) entry of those dependencies (e.g. `rettime`, `@open-draft/deferred-promise`), even for deps that also ship a CJS `main`.
  - jest-expo's default `transformIgnorePatterns` is `/node_modules/(?!(.pnpm|react-native|…))`. pnpm nests real packages at `node_modules/.pnpm/<pkg>/node_modules/<pkg>/…`; the **inner** `/node_modules/` re-triggers the negative lookahead, so the `.pnpm` whitelist never reaches these nested packages and their `.mjs` is left untransformed → `SyntaxError: Unexpected token 'export'`.
  - The only fixes are resolver-condition or `transformIgnorePatterns` changes, which live **only** in the shared jest config — the exact block the plan forbids touching. There is no per-test-file jest override for either. Mocking each ESM dependency (I verified `jest.mock('rettime', …)` clears the first failure but the next ESM dep — `@open-draft/deferred-promise` — immediately surfaces) is an unbounded, fragile chain that would make the test pass for the wrong reasons, so I did not ship it.
  - **What I did instead:** left `api.test.ts` written to the R7 spec (four assertions, `msw` `setupServer`), passing `tsc` and `eslint`, but not green under `jest`. The production module `app/core/services/api.ts` is unaffected and fully passes typecheck + lint. **This is raised as an ESCALATION** — the fix requires a decision on the shared jest config that is out of this ticket's stated bounds. Options for the verifier / a follow-up: (a) narrow `transformIgnorePatterns` to also match nested pnpm paths for msw's ESM deps, or (b) add `testEnvironmentOptions.customExportConditions` (or resolver conditions) so msw's deps resolve to their CJS entries — both are one-line edits to `app/package.json`'s jest block that the plan explicitly ruled off-limits.

- **Minor, within scope:** added a private `ApiConfigError` class (used by `resolveBaseUrl`) so the "env unset" failure is a typed throw rather than a bare `Error`; this is an implementation detail of plan step 3 (R2), not a new requirement.
