- **Anything the verifier should pay special attention to:**
  - The sibling-vs-nested-Pressable composition in `ActivityList.tsx`'s `ActivityRowItem`: `JobCard` and the remove `Button` are siblings inside a `View`, not nested inside `JobCard`'s `Card` `Pressable`. This was mutation-tested during implementation — temporarily wiring both `JobCard.onPress` and `Button.onPress` to fire `onPress` **and** `onRemove` together caused both of `ActivityList.test.tsx`'s independence assertions to fail as expected, then the code was restored to the correct (verified byte-identical via `diff`) sibling implementation. Worth an independent look to confirm the restored file is indeed what's committed.
  - `applied.tsx`'s `handlePress`/`favorites.tsx`'s `handlePress` use the `index` argument `ActivityList`'s `FlatList` `renderItem` already provides, rather than re-deriving it via `findIndex` — avoids an O(n) lookup per press and matches `index.tsx:254-256`'s existing `handleJobPress(job, index)` precedent.
  - `Button`'s `accessibilityLabel` prop (e.g. `"Cancelar Backend Engineer"`) is used both for a11y and as the test query key (`getByLabelText`) — distinguishes the remove action from same-titled buttons across rows without adding a new prop to `Button` itself (uses its existing `PressableProps` passthrough).
