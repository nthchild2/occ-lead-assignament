## Deviations from the plan

- **Plan step 8 (typecheck/lint/test pass) → added `react-test-renderer@18.3.1` as an explicit `app/package.json` devDependency, not in the plan's change list.** Reason: this repo has no prior `renderHook`/`act`-based test (the plan's own Risks section flagged this as "First async-hook test pattern in the codebase"). Without a direct devDependency pin, pnpm's resolver hoisted a transitive `react-test-renderer@19.1.0` (pulled in by `jest-expo`/`@testing-library/react-native`) that is incompatible with this workspace's `react@18.3.1`, causing `core/hooks/useJobs.test.ts` and `core/hooks/useDebounce.test.ts` to fail to even load (`TypeError: Cannot read properties of undefined (reading 'S')` inside `react-test-renderer`'s internals) — a pure dependency-resolution problem, not a logic issue in any plan-authored file. This was necessary to make plan steps 6 and 7 (the two `renderHook`-based tests the plan itself requires) actually runnable, so it was made rather than escalated. Scope was kept minimal: one devDependency line pinned to the exact version already used elsewhere in the workspace (`react@18.3.1`), plus the resulting `pnpm-lock.yaml` update — no other dependency, script, or config changed. **Correction (per VERIFY):** the implementer's original claim that `pnpm why react-test-renderer` shows a single version "both before and after the pin" was independently checked by the verifier and found inaccurate for "after" — `pnpm why` shows two coexisting versions post-pin: `18.3.1` (used by `@occ/app`/`@testing-library/react-native`, resolving the incompatibility this fix targets) and `19.1.0` (still pulled directly by `jest-expo`'s own dependency graph, unaffected by and unrelated to this pin). The fix itself is still correct and scoped to the one devDependency line plus its lockfile cascade — pnpm's per-package dependency resolution means `@occ/app`'s own `react-test-renderer` now correctly resolves to `18.3.1`, which is what made the tests runnable; the coexisting `19.1.0` belongs to a different package's dependency tree and doesn't affect this workspace.
- Everything else in the plan's change list (steps 1–7) was implemented as specified. **No other deviations.**
