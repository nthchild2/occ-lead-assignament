## Approach

Build the four new files in dependency order — service, then store, then hooks, then tests — so every intermediate state type-checks against real `@occ/shared` types (`Job`, `JobFilters`, `Pagination`, `JobListResponse`). `jobs.service.ts` is a thin wrapper mirroring `auth.service.ts`'s shape, with a small local `buildQueryString` helper since `get(path, schema, auth?)` only accepts a pre-built path string (no query-object overload exists). `jobs.store.ts` mirrors the plain, non-persisted `create<T>((set, get) => ({...}))` form of `theme.store.ts` — rejected the curried `persist(...)` form of `auth.store.ts` because A2 Decision 1 explicitly excludes jobs from persistence and the store must stay reset-on-launch. `useJobs.ts` owns all async orchestration and error handling (per `docs/MAP.md`'s no-fetch-in-stores rule); `useDebounce.ts` is a fully generic, store-agnostic hook with a caller-supplied delay, kept separate because it has no dependency on jobs domain state and is reusable by the future search screen. Rejected alternative: folding debounce logic directly into `useJobs` — rejected because R7's acceptance criterion requires a standalone generic hook, and coupling it to jobs state would block reuse by the (out-of-scope) search screen ticket. Where A2's illustrative snippets drift from the spec/real schema (narrower 3-field `pagination` type, `useJobs` omitting `fetchPage` from its return), the plan follows the spec (R2, R6) and the real `@occ/shared` `Pagination` type, per research's flagged (non-blocking) drift.

## Risks & rollback

- **Query-string builder correctness** (change #1): if `buildQueryString` mis-serializes filter keys (must match `JobFiltersSchema`'s field names verbatim — `q`, `city`, `salary_min`, `salary_max`, `sort`, `page`, `limit` — per `backend/src/domains/jobs/jobs.router.ts:15-21`), requests silently return unfiltered/wrong data. Contained entirely within `jobs.service.ts`; rollback is reverting that one file, no downstream coupling since `useJobs.ts` only calls `jobsService.list(filters)`.
- **`Pagination` shape mismatch** (change #2): using the real 5-field `@occ/shared` type instead of A2's stale 3-field sketch is a deliberate deviation from A2's illustrative snippet. If a reviewer expected the narrower shape, this is a one-line type-import fix, not a structural rollback, since the store's initial state and `appendJobs` signature are the only touch points.
- **In-flight guard relying solely on `isLoading`** (change #4): if a future caller mutates `isLoading` outside `useJobs` (e.g., directly via `setLoading`), the guard could be bypassed. Low risk since `MAP.md` mandates components call actions, never raw setters, and no other consumer exists yet in this ticket's scope. Rollback: revert `useJobs.ts` only.
- **First async-hook test pattern in the codebase** (change #6): no existing precedent for combining `renderHook`/`act` with `global.fetch` mocking here; if the pattern proves flaky (act warnings, timer/microtask ordering), the fix is local to the test file and does not affect `useJobs.ts`'s implementation.
- General rollback strategy: all seven changes are additive new files (no edits to existing shared/service/store files), so any single change can be reverted independently by deleting its file(s) without touching the other six.
